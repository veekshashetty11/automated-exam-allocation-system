Automated Exam Allocation System

Conflict-aware, multi-classroom, multi-slot exam scheduling

About the Project

This project is a Data Structures and Algorithms (DSA) based system designed to allocate students to examination seats and invigilators while handling multiple real-world constraints.

Key Features

  • Multiple classrooms and seating capacity handling
  • Morning & Afternoon time-slot scheduling
  • Conflict-aware seat allocation
  • Question paper set distribution
  • Subject-per-classroom limit
  • Late student handling using Queue
  • Invigilator load balancing with absence handling

Data Structures Used

  • Arrays & Vectors
  • Queue (Late students)
  • Hash Map (Subject limits)
  • Priority Queue (Invigilator assignment)

Workflow

  1. Input classrooms and capacity
  2. Input students and subjects
  3. Assign question sets
  4. Allocate seats per classroom & time slot
  5. Handle late students
  6. Assign invigilators fairly
